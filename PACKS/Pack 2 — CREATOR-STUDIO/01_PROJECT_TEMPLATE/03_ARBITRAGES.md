# 03 — Arbitrages

Document les choix qui ferment des options pour ce projet.  
Un arbitrage = une décision prise + la raison. Pas de retour en arrière sans nouvelle entrée.

---

## Pourquoi documenter les arbitrages ?

Sans ce fichier, on refait les mêmes questions à chaque itération :  
"Et si on ajoutait de la batterie ?" — "Et si on rendait ça plus pop ?"  
Les arbitrages ferment ces questions pour permettre d'avancer.

---

## Format standard

```
[Date] — [Choix fait] — [Raison]
```

---

## Exemples remplis

**Direction vocale**
```
2026-04-10 — Vocals sparses uniquement (pas de voix lead) — L'intimité du projet prime sur la force du hook. Une voix lead déplacerait le centre de gravité du track vers une logique chanson commerciale, ce qui n'est pas l'objectif.
```

**Percussion**
```
2026-04-11 — Pas de percussion lourde — Le mood nocturne défini dans 01_BRIEF.md est incompatible avec une batterie full. On garde des percussions légères ou absentes selon le track.
```

**BPM**
```
2026-04-12 — BPM fixé entre 82 et 88 — Plage testée sur 3 générations. En dehors de cette plage, le groove devient trop lourd ou trop nerveux pour le mood cible.
```

**Polish**
```
2026-04-14 — Pas de son "trop propre" — Le projet assume une esthétique lo-fi. Les générateurs en mode "polished" ou "mastered" seront évités ou re-traités.
```

## Règles d'usage

1. Un arbitrage **ne se supprime jamais** — il peut être contredit par un nouvel arbitrage avec date.
2. Si vous revenez sur un choix, ajoutez une nouvelle entrée avec la raison du revirement.
3. Ce fichier est lu avant chaque session de génération pour éviter les allers-retours.
