# Reference Translation - Pro

Comment transformer plusieurs references artistiques en directives exploitables sans imitation.

## 1) Principe

Une reference sert a decrire une intention, pas a copier une oeuvre.

Objectif: traduire les references en parametres abstraits:

- energie
- densite
- texture
- trajectoire emotionnelle
- lisibilite rythmique

## 2) Methode en 4 etapes

### Etape 1 - Decomposer chaque reference

Pour chaque reference, noter:

- Ce que je garde
- Ce que je rejette
- Le risque principal de derive

### Etape 2 - Fusionner les points communs

Identifier 3 a 5 invariants partages entre references.

Exemple:

- groove stable
- texture sombre mais lisible
- progression sans explosion festival

### Etape 3 - Resoudre les contradictions

Quand deux references s'opposent, trancher explicitement:

- priorite rhythmique
- priorite harmonique
- priorite de mix

Ecrire la decision en une phrase actionnable.

### Etape 4 - Convertir en contraintes prompt

Generer:

- contraintes positives (ce qu'on veut)
- contraintes negatives (ce qu'on exclut)
- KPI d'ecoute (ce qui valide)

## 3) Politique de non-imitation

- Interdit: demander une copie d'un artiste, titre ou hook identifiable.
- Autorise: decrire des qualites generales (texture, energie, structure).
- Obligation: ajouter une signature propre au projet (motif, arc, arrangement).

## 5) Quality gate

La traduction est valide si:

- Les references ne sont plus citees mot a mot dans le prompt final
- Les decisions d'arbitrage sont explicites
- Les exclusions couvrent les risques de copie/derive
- La sortie conserve une identite originale
