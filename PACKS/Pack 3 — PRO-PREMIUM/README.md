# Pro Studio System

Ce pack formalise un workflow de niveau studio: direction, iteration, debugging et coherence multi-projets.

## Pivots de valeur

- taxonomie avancee de styles
- playbook de debugging
- standards de versioning et release
- deep case studies complets
- bundle tool-specific possible
- TOOLS_START_HERE.md: guide d'outils pour pipeline pro et arbitrages multi-projets
- AI_STUDIO_AND_DISTRIBUTION.md: cadrage pro des stacks IA + distribution multi-canaux

## Comment utiliser ce pack

1. Formalisez direction artistique : complétez STYLE_TAXONOMY_ADVANCED (stable vs risky descriptors).
2. Structurez le prompt avec NAMING_VERSIONING_STANDARD (v01_base, v02_variant, RC1, FINAL).
3. Assemblez le prompt principal, variantes outils, et contraintes (section 6 de votre fiche).
4. Generez et ecoutez : formez hypothese testable dans ITERATION_LOG.
5. Enregistrez resultat + criteres d'acceptation par version.
6. Si anomalie detectee : diagnostiquez avec DEBUG_PLAYBOOK (symptom -> correction).
7. Une fois validee, versionnez RC1 et preparez export FINAL.
8. Documentez releaseability : checklist RELEASE_WORKFLOW, metadata sync/distribution.
9. Standardisez pipeline dans TOOLS_START_HERE selon votre outil principal (Suno, Udio, etc.).
