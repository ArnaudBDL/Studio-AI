# Quality Rules

## Prompt rules

- 1 dominant intention per prompt
- 2 to 5 style markers max
- 1 structural instruction that can be verified after listening
- 1 to 3 constraints max

## Workflow rules

- Document each meaningful iteration
- Do not rename versions casually
- Save failed attempts if they teach something useful

## Quality gate

Un prompt passe si:

- il est explicable en 20 secondes
- la modification par rapport a la version precedente est claire
- le resultat attendu peut etre observe a l'ecoute
