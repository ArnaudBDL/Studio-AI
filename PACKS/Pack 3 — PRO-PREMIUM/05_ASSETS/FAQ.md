# FAQ — Pack 3 PRO-PREMIUM

Questions fréquentes des acheteurs du Pack 3.  
Pour les questions sur les packs FREE/STARTER/CREATOR, voir les FAQ respectives.

---

## Général

**Q : Quelle est la différence entre le Pack 2 Creator et le Pack 3 Pro ?**  
R : Le Pack 2 couvre un projet type de A à Z avec des mini-cas pratiques. Le Pack 3 va plus loin avec des pipelines complets documentés (brief → prompt → iterations → release), des add-ons outil-spécifiques (Suno, Udio), et une taxonomie stylistique avancée pour des workflows multi-projets professionnels.

**Q : Faut-il acheter les autres packs avant le Pack 3 ?**  
R : Non, le Pack 3 est autonome. Il contient ses propres bases. Cela dit, si vous débutez avec l'IA musicale, le Pack 1 ou 2 est un meilleur point de départ.

**Q : Les méthodes sont-elles valables longtemps ? Les outils évoluent vite.**  
R : Les principes de construction de prompt (style cluster, mood, texture, contraintes) sont stables car ils reflètent comment les modèles sont entraînés, pas juste l'interface d'un outil. Les notes outil-spécifiques (Suno, Udio) sont mises à jour à chaque version majeure.

## Utilisation

**Q : Par où commencer si j'ai déjà un projet en cours ?**  
R : Allez directement dans `01_PROJECT_TEMPLATE_PRO/README.md` — il structure un projet existant en 10 minutes.

**Q : Je génère beaucoup de tracks mais j'ai du mal à maintenir une cohérence stylistique sur un EP. Ce pack m'aide ?**  
R : C'est exactement pour ça que le Pack 3 existe. Le `PROMPT_MATRIX.md`, le `STYLE_TAXONOMY_ADVANCED.md` et les deep case studies sont conçus pour ça. Voir aussi `ITERATION_LOG.md` pour le suivi des versions.

**Q : Je veux utiliser ce pack avec Udio, pas Suno. C'est possible ?**  
R : Oui. Le cœur du Pack 3 est outil-agnostique. L'add-on Suno est un bonus. Les mêmes méthodes de prompting s'appliquent à Udio avec des ajustements mineurs (voir `AI_STUDIO_GUIDES/` pour les guides par outil).

**Q : Est-ce que les prompts fonctionnent en français ?**  
R : Les outils IA musicaux (Suno, Udio, Soundraw) sont entraînés majoritairement sur des données en anglais. Les prompts en anglais donnent des résultats plus stables. Tous les templates du Pack 3 sont en anglais pour cette raison. La documentation et les guides sont bilingues.

## Technique & Outils

**Q : Suno change souvent d'interface. Les templates sont-ils à jour ?**  
R : Les templates sont rédigés pour Suno v3/v3.5/v4. Les comportements fondamentaux (traitement du style cluster, contraintes vocales, longueur optimale) restent stables entre versions. Si une interface change, reportez-vous aux `SUNO_PROMPTING_NOTES.md` pour les ajustements.

**Q : J'ai un résultat correct mais pas parfait. Comment itérer efficacement ?**  
R : Utilisez `SUNO_FAILURE_MODES.md` (ou l'équivalent pour votre outil) pour diagnostiquer le problème, puis documentez chaque tentative dans `ITERATION_LOG.md`. La méthode : changer un seul paramètre à la fois, évaluer, noter.

**Q : Est-ce que je peux utiliser les tracks générées commercialement ?**  
R : Ça dépend des CGU de chaque outil. Suno Pro et Udio Pro autorisent un usage commercial. Vérifiez toujours la licence de l'outil au moment de l'export. Ce pack ne modifie pas les droits d'usage des outils tiers.

## Contenu & Mises à jour

**Q : Le pack inclut-il des exemples audio ?**  
R : Non — des fichiers audio alourdirait le pack et deviendraient obsolètes rapidement. Les exemples sont des prompts documentés avec description du résultat attendu (voir `03_DEEP_CASE_STUDIES/`).

**Q : Comment sont structurées les mises à jour ?**  
R : Voir `CHANGELOG.md` pour l'historique. Les mises à jour majeures (nouvelle version d'outil, nouveau deep case, correction) sont versionnées. Les acheteurs reçoivent les mises à jour gratuitement.

**Q : Je veux contribuer un cas ou signaler une erreur. Comment ?**  
R : Contactez via la plateforme d'achat. Les meilleures contributions sont intégrées dans les mises à jour avec mention (anonyme ou nominative selon préférence).
