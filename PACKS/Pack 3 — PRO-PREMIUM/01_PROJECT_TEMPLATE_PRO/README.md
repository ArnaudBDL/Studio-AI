# Project Template Pro

Template de projet Pack 3 pour conduire une prod complete, du brief a la release.

## 1) Structure recommandee

Creer un dossier projet avec cette structure:

- 01_BRIEF.md
- 02_REFERENCES.md
- 03_ARBITRAGES.md
- 04_PROMPTS.md
- 05_ITERATIONS.md
- 06_EXPORT_RELEASE.md

## 2) Workflow standard

1. Brief: objectif business + objectif artistique + contraintes
2. References: decomposition et translation non-imitative
3. Arbitrages: decisions en cas de conflit
4. Prompting: v1 puis iterations ciblees
5. QA: verification selon checklist qualite
6. Export: versions finalisees + metadata
7. Release: nommage, packaging, distribution

## 3) Variantes par type de track

### Club / DJ
- Intro/outro longues et mixables
- Focus groove/kick/bass
- Transitions lisibles en set

### Streaming pop/electro
- Hook plus rapide
- Duree plus compacte
- Clarte vocale/lead prioritaire

### Cinematic / sync
- Arc narratif plus marque
- Evolution texture/dynamique
- Contraste sections accentue

## 4) Checklist projet (minimum)

- [ ] Brief valide et testable
- [ ] References traduites en contraintes
- [ ] Arbitrages ecrits explicitement
- [ ] Prompt final versionne
- [ ] 2 ecoutes de validation croisee
- [ ] Export WAV/MP3 nommes proprement
- [ ] Metadata completees

## 5) Naming convention

`PROJ_STYLE_BPM_v01`

Exemple:

`NIGHTDRIVE_DARK-HOUSE_125_v01`

Increment:

- `v01` -> `v02` pour changements de fond
- `v02a` pour micro-correction
- `FINAL` uniquement apres validation checklist

## 6) Definition of done

Le projet est termine si:

- Le prompt final est stable et reproductible
- Le rendu est coherent avec le brief
- Les exclusions sont respectees
- Les assets release sont prets
